GnuCash 2.4.0 is out ,mysql ,postgresql,sqlite

Free software General

This has been a refreshing change for me.GnuCash has been a very good personal accounting tool for me and I have enjoyed using it whenever needed.I have recommended it t a couple of people as well.

The new feature which i am thrilled to use is connectivity to the database ( GDA ).This allows you to store the gnucash data in backend tables.You can use sqlite,mysql or postgresql to store the data. Most of the gnucash business logic is still in the frontend and it would take some time for the complete makeover as I understand.

Using the database feature is pretty straight forward.If the option is compiled in your gnucash binary you will notice xml,mysql,postgresql as an option available when you do a “Save As” in the file menu.

[caption id=“attachment_64” align=“alignnone” width=“300” caption=“GnuCash database options”] [/caption]

Once you have the data in the database it is really a matter of connecting and getting the reports in the formats you desire. One limitation in the earlier versions was the need to understand “guile” to write reports for gnucash.But now it is just a matter of connecting to the backend and writing the reports.Come on jasperreports…

Major changes in the 2.4.0 release include; 

      * In addition to the XML backend, Gnucash can now use a SQLite3,
        MySQL or PostgreSQL database to store the data. This is a new
        implementation using libdbi. It supports all features including
        the business features. In order to build with this, add
        --enable-dbi to the configure command. In addition to the
        libdbi-dev package for your distribution, you will also need the
        appropriate DBD (libdbi driver) package for sqlite3, mysql or
        postgresql.
      * As a replacement for the current GtkHTML HTML engine used to
        display reports and graphs, Gnucash can use WebKit. WebKit is
        the engine used by Google Chrome on Windows and Safari on Apple.
        In order to build with this, add --with-html-engine=webkit to
        the configure command. You will need an appropriate webkit-dev
        package. On win32, you will need to download the
        webkit-1.1.90-win32.zip file from the source repository and put
        it into the downloads directory of your gnucash build area.
      * Updated AqBanking on Win32.
